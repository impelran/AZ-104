<h1>1. Prérequis</h1>
<h2>🌞 Déterminer quel algorithme de chiffrement utiliser pour vos clés:</h2>
• OpenSSH a déclaré ssh-rsa obsolète.
• L'entreprise SSH dérière le protocol SSH préconise d'utiliser ed25519 étant le dernier algo mis en place et Azure n'authorise uniquement RSA ou Ed25519.

<br>

<h2>🌞 Générer une paire de clés pour ce TP:</h2>
• ssh-keygen -f .\.ssh\cloud_tp1_key -t ed25519
• Utilisation d'une passphrase

<br>

<h2>🌞 Configurer un agent SSH sur votre poste:</h2>
• Utilisation de l'agent SSH directement dans le PowerShell

<br>
<br>

<h1>2. Spawn des VMs</h1>
<h2>🌞 Connectez-vous en SSH à la VM pour preuve:</h2>
• .ssh\cloud_tp1_key.pub<br>
• Création d'une VM:<br>
    • Subscription: Subscription<br>
    • Resource group: Main<br>
    • Virtual machine name: TP1<br>
    • Region: (Europe) France Central<br>
    • Availability options: No infrastructure redundancy required<br>
    • Image: Ubuntu Server 24.04 LTS<br>
    • Authentication type: SSH public key<br>
    • Username: azureuser<br>
    • SSH public key source: Use existing public key<br>
    • SSH public key: ssh-ed25519...<br>
    • Public inbound ports: Allow selected ports<br>
    • Select inbound ports: SSH (22)<br>
• ssh -i .ssh/cloud_tp1_key azureuser@4.211.156.221

<br>

<h2>🌞 Créez une VM depuis le Azure CLI:</h2>
• Installation de Azure CLI<br>
• az login<br>
• az group create --name TP1 --location francecentral<br>
• az vm create -g TP1 -n TP1vm --image Ubuntu2204 --admin-username azureuser --ssh-key-values ~/.ssh/cloud_tp1_key.pub --size Standard_D2s_v3<br>
• Résultat:<br>
